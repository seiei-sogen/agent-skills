# 箇条書き化推敲

日本語の文章を、意味・トーン・情報量を保ったまま読みやすく推敲する。
特に、列挙・条件・手順・注意点・要件・比較・判断基準が一文に詰め込まれている箇所を、必要に応じて箇条書きへ再構成する。

## 基本原則

- 「長いから箇条書き」ではなく、「並列情報だから箇条書き」と判断する。
- 文章全体をむやみに箇条書き化しない。
- 読者が情報を整理しやすくなる箇所だけを箇条書きにする。
- 意味を追加・削除・誇張しない。
- 元の文体が硬い場合は硬めに、カジュアルな場合はカジュアルに保つ。
- 箇条書き化によって文の流れが悪くなる場合は、通常の文のまま整える。

## 作業手順

1. 原文の用途、読者、文体、Markdown などの既存構造を確認する。
2. 一文に複数の列挙・条件・手順・注意点・要件・比較・判断基準が含まれていないか確認する。
3. 箇条書き化する条件と、箇条書き化しない条件の両方に照らして判断する。
4. 箇条書き化する場合は、短い導入文を置き、各項目の文法形式をそろえる。
5. 推敲後に、否定・条件・例外・因果関係・技術的意味が変わっていないか確認する。

## 箇条書き化を検討する条件

次のいずれかに当てはまる場合、箇条書き化を検討する。

- 3項目以上の列挙がある。
- 2項目でも、それぞれが長い、または説明・条件・理由・例を含む。
- 一文が長く、複数の論点が読点で連結されている。
- 「A、B、C」の各要素が長く、読者が区切りを見失いやすい。
- 手順、要件、注意点、メリット・デメリット、判断基準、例外条件が含まれる。
- 読者が後で参照しそうな情報である。
- 「〜すること」「〜の場合」「〜ため」「〜によって」などが複数並ぶ。
- 「〜し、〜し、〜する」のように、1つの文に複数の動作・条件・対象が含まれている。

## 箇条書き化しない条件

次の場合は、無理に箇条書きにしない。

- 短い単語や名詞句の単純な列挙で、横並びの方が自然である。
- 箇条書きにすると、文の勢い、余韻、自然さが失われる。
- 物語文・感情表現・エッセイ的な文章で、流れを切ると弱くなる。
- 列挙に見えても、実際には一つの熟した表現やまとまった概念である。
- 箇条書き化によって、元の文章より堅苦しくなる。
- 箇条書きにすると、かえって大げさ・事務的になる。

## 変換ルール

箇条書き化する場合は、次の形式を優先する。

1. 箇条書きの前に、短い導入文を置く。
2. 各項目は同じ文法形式でそろえる。
3. 1項目につき1情報を原則とする。
4. 項目数は原則3〜7個に収める。
5. 各項目が長くなる場合は、親項目と子項目に分ける。
6. 項目末尾は、文章の硬さに応じて「。」を付けるか省略する。
7. 元の文章にない見出しは、必要最小限にする。
8. 箇条書きにした後、前後の文との接続を自然に整える。

## 補助的な目安

文字数だけで判断しない。
ただし、次は箇条書き化や文分割を検討する補助目安にする。

- 列挙部分の合計が40文字以上である。
- 各項目の平均が15〜20文字以上である。
- 一文が80文字以上である。
- 読点が3つ以上続き、論点が分かれている。
- 読点で区切られた各部分が、それぞれ独立した意味を持つ。
- 列挙項目が長い、または各項目が独立した意味を持つ。

## 代表的な変換パターン

### 要件列挙

Before:

```text
この機能を利用するには、Aであること、Bしていること、Cが完了していることが必要です。
```

After:

```text
この機能を利用するには、次の条件を満たす必要があります。

- Aであること
- Bしていること
- Cが完了していること
```

### 注意点列挙

Before:

```text
利用時は、保存前に内容を確認し、不要な個人情報を削除し、公開範囲を確認してください。
```

After:

```text
利用時は、次の点に注意してください。

- 保存前に内容を確認する
- 不要な個人情報を削除する
- 公開範囲を確認する
```

### 比較

Before:

```text
Aは導入が簡単で小規模向きですが、Bは設定が複雑な一方で大規模運用に向いています。
```

After:

```text
AとBの違いは次のとおりです。

- A：導入が簡単で、小規模向き
- B：設定は複雑だが、大規模運用向き
```

## 出力方針

- 原則として、推敲後の文章だけを出力する。
- ユーザーが理由を求めた場合のみ、どこをなぜ箇条書き化したかを簡潔に説明する。
- 原文の意味を変えた可能性がある場合は、その旨を注記する。
- ファイルを編集した場合は、変更した箇所と、判断に迷った箇所だけを簡潔に報告する。
