# Git ブランチ名を提案する

入力から変更の主目的を特定し、コピーして使える Git ブランチ名を提案する。
ブランチの作成、切り替え、名前変更は、ユーザーが明示的に求めない限り行わない。

## 入力を確認する

ユーザーから次のいずれかを受け取る。

- 変更内容、issue、仕様、作業メモなどの文章
- ローカルファイルのパス

入力が読み取り可能なファイルのパスなら、パス、ファイル名、内容を読み取る。
ファイルは変更しない。
パスが存在しない、または読み取れない場合は、見えているパス要素だけを根拠にする。
パスだけでは変更の目的が分からない場合は、内容または目的の提示を求める。

## 手順

1. 入力から、対象、変更内容、目的を抽出する。
2. 複数の変更が含まれる場合は主目的を選ぶ。独立した変更に分けるべき場合は、その旨を短く伝える。
3. リポジトリに明示されたブランチ命名規則があれば、それを優先する。
4. 明示された規則がなければ、`<type>/<slug>` 形式で候補を作る。
5. 候補が Git のブランチ名として有効か確認する。
6. 最も具体的で簡潔な候補を第一候補として出力する。

## Type を選ぶ

明示された規則がない場合は、次の type から変更の主目的に最も近いものを選ぶ。

| type | 用途 |
| --- | --- |
| `feat` | 新しい機能や利用者向けの振る舞いを追加する |
| `fix` | 不具合を修正する |
| `docs` | 文書だけを追加・修正する |
| `refactor` | 外部の振る舞いを変えずに実装を整理する |
| `test` | テストだけを追加・修正する |
| `perf` | 性能を改善する |
| `ci` | CI の設定や処理を変更する |
| `build` | ビルド処理やビルド依存関係を変更する |
| `chore` | 上記に当てはまらない保守作業を行う |

入力が曖昧だからという理由だけで `chore` を選ばない。
変更種別を判断できない場合は、推測で埋めずに確認する。

## Slug を作る

- 英語の小文字とハイフンを基本にする。
- 日本語はローマ字へ逐語変換せず、意味を保った短い英語にする。
- `add-user-search` や `prevent-refresh-token-loop` のように、変更を表す動詞と対象を優先する。
- 2〜6語、ブランチ名全体で60文字以内を目安にする。
- `change`、`update-stuff`、`misc` など、内容を特定できない語を避ける。
- 入力にある issue ID やチケット番号は改変・捏造しない。リポジトリ規則があれば、その位置と表記に従って含める。
- OAuth、CLI、TSDoc などの技術語は、識別できる小文字表記にする。
- ユーザー名、issue ID、対象範囲など、入力にない情報を補わない。

## Git の制約を確認する

候補に次が含まれないことを確認する。

- 空白、制御文字、`~`、`^`、`:`、`?`、`*`、`[`、`\`
- `..`、`@{`、連続する `/`
- 先頭または末尾の `/` や `.`
- 末尾の `.lock`

候補を `-` で始めない。
Git を利用できる場合は、候補を引数にして検証する。

```bash
git check-ref-format --branch 'feat/add-user-search'
```

## 出力する

通常は第一候補を1件だけ、インラインコードで出力する。

```markdown
`feat/add-user-name-search`
```

変更種別や表現に意味のある選択肢がある場合だけ、第一候補に加えて代替候補を最大2件示し、違いを一言で説明する。
入力から対象や目的を特定できない場合は、汎用的な名前を作らず、判断に必要な質問を1つだけする。
ブランチ作成コマンドや長い分析過程は、求められない限り出力しない。

## 例

| 入力 | 第一候補 |
| --- | --- |
| `ユーザー一覧に氏名検索を追加する` | `feat/add-user-name-search` |
| `ログアウト後もセッションが残る不具合を修正する` | `fix/clear-session-on-logout` |
| `README のローカル環境構築手順を更新する` | `docs/update-local-setup-guide` |
| `認証処理を整理して読みやすくする` | `refactor/simplify-auth-flow` |
| `issues/fix-refresh-token-loop.md` | `fix/prevent-refresh-token-loop` |
