# My Coding Rule レビュー

`MY_CODING_RULE.md` に基づき、過去のレビュー指摘を繰り返さないためのセルフレビューを行う。

---

## 対象スコープの決定

| 引数 | スコープ |
|------|---------|
| 引数なし | `git diff main...HEAD --name-only` で変更ファイル一覧を取得 |
| ディレクトリパス | 指定パス配下をレビュー |

- 引数: `$ARGUMENTS`

---

## 手順

### 1. ルールの読み込み

`MY_CODING_RULE.md` を Read で読み、全ルールを把握する。

### 2. 対象ファイルの読み込み

スコープ内の **全ファイルの全内容** を読み込む。

### 3. ルール別チェック

以下のカテゴリ順に、各ルールへの違反を検出する。

#### カテゴリ 1: 関数・モジュールの独立性

| ルール | チェック観点 |
|--------|------------|
| 呼び出し側の実装に依存しない | 関数が呼び出し側の内部実装を前提としていないか |
| 外部の順序保証に依存しない | API レスポンスの並び順に依存していないか |
| 純粋な変換ロジックはContainerに書かない | フィルタ・ソート等が Container にインラインで書かれていないか。`_utils/` に切り出すべきか |
| 不要な条件を含めない | ロジックの目的に無関係なフィルタ条件が混ざっていないか |

#### カテゴリ 2: コードの衛生

| ルール | チェック観点 |
|--------|------------|
| コメントアウトではなく削除する | コメントアウトされたコードが残っていないか |
| コメントにチケット番号を残さない | コメントに `PFM-xxx` 等のチケット番号が含まれていないか |
| JSX内にインライン関数を定義しない | `onClick` 等のハンドラが JSX 内でインライン定義されていないか |
| React Compiler有効時は手動メモ化しない | 手動の `useMemo` / `useCallback` / `React.memo` が使われていないか |
| import は named import で統一する | `React.ChangeEvent` のような名前空間経由の参照がないか |
| 処理は使う場所で宣言する | 変数や関数が不必要に遠い場所で宣言されていないか |

#### カテゴリ 3: PR・コミットの作法

| ルール | チェック観点 |
|--------|------------|
| 無関係な変更はPRを分ける | 機能変更と無関係なファイル変更が含まれていないか |
| 設計ドキュメントをコミットに含めない | 設計ドキュメントがコミットに含まれていないか |
| 意図しないファイル変更・削除に注意 | 差分に意図しないファイルの削除・変更がないか |

#### カテゴリ 4: 可読性

| ルール | チェック観点 |
|--------|------------|
| 非自明な意図にはコメントを付ける | 複雑なロジックに「なぜ」の説明がないか |
| コールバック引数は略語を避ける | `e` / `ev` のような略語が使われていないか |
| マジックナンバーは定数化する | JSX 内に数値リテラルが直接書かれていないか |
| ピクセル固定値は8の倍数にする | CSS の `width` / `height` のピクセル固定値が 8 の倍数でないか |
| 関連する変数は近くに配置する | 関連する宣言やロジックが物理的に離れていないか |

#### カテゴリ 5: 新規作成の前に

| ルール | チェック観点 |
|--------|------------|
| 既存のものを確認してから作る | 新規追加されたユーティリティ・コンポーネントに、既存の類似品がないか |

#### カテゴリ 6: テスト

| ルール | チェック観点 |
|--------|------------|
| 純粋関数にはテストを書く | `_utils/` に追加された純粋関数に対応する `.test.ts` が存在するか |

### 4. 報告

違反が見つかった場合、以下の形式で報告する。違反がなければ「違反なし」と報告する。

---

## 出力形式

### 違反あり

```
### [カテゴリ名] ルール名
- **ファイル**: `path/to/file.ts:行番号`
- **違反**: 具体的な問題の説明
- **修正案**: 提案する修正内容
```

### サマリー

最後に以下のサマリーを付与する。

```
## サマリー
- 違反: N 件
- カテゴリ別: 独立性 N / 衛生 N / PR N / 可読性 N / 新規作成 N / テスト N
```

---

## 注意事項

- このスキルはコードを **修正しない**。報告のみに留める。
- `MY_CODING_RULE.md` の内容が更新されている可能性があるため、毎回必ず Read で最新の内容を読み込むこと。
- プロジェクト規約（`docs/conventions/`）との重複ルールは、このスキルでは `MY_CODING_RULE.md` の観点のみでチェックする。プロジェクト規約の網羅的チェックは `/frontend-review` や `/backend-review` に任せる。
